What's in the box

🧭

DAG-teams orchestration

Phase-1 LLM planner → Phase-2 deterministic scheduler. Every node has a verifiable output.

🧠

Progressive-disclosure memory

Three-tier memory with ≥5× token reduction on long research runs.

📚

Citation-graph KB

SQLite + FTS5, weighted edges, multi-hop synthesis, native arXiv / GitHub / HuggingFace adapters.

🛡️

5-tier verifier

Lexical → mutation → LLM-judge → executable → cross-channel, plus a symbolic claim-number tier.

🧪

9-specialist cohort

ResearchDirector, Methodologist, ThreatModeler, Publisher + 5 others. Optional subprocess isolation.

🔌

7-tool MCP server

Read-only: skill.*, kb.*, memory.* — drop-in parity with claude-mem.

🧬

Self-wiring graph

Zero-LLM regex citations, typed-edge cascade, entity expansion, stale-link reconciler, backlink-boosted ranking.

📦

Tool-output sandbox

Context Mode pattern — FTS5-indexed payload deferral keeps the context window hot, not bloated.

⚡

Chaos + red-team

Fault injection, 4 security probes, a HAFC-lite primitive-level failure attribution matrix.

Quickstart

Local dev

git clone https://github.com/ndqkhanh/open-fang.git
cd open-fang
make install        # venv + harness_core + deps
make test           # 612 tests (1 network-deselected)
make lint           # ruff
make run            # uvicorn on :8010

CLI

openfang skill list
openfang mcp serve                     # stdio MCP (7 read-only tools)
openfang mcp import manifest.json
openfang trace validate trajectories.jsonl
